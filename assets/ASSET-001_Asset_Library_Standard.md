# ASSET-001
# Project Orion Asset Library Standard

Version: 1.0
Status: Baselined

---

## 1. Purpose

This document defines the organization, naming conventions,
and maintenance standards for the Project Orion Asset Library.

The Asset Library serves as the single source of truth for
all reusable project media.

---

## 2. Scope

The standard applies to assets referenced by:

- GitHub documentation
- Project website
- GitHub Releases
- LinkedIn publications
- Engineering documentation
- Presentations

---

## 3. Asset Library Structure

assets/
├── architecture/
├── branding/
├── disaster-recovery/
├── github/
├── hardware/
├── network/
├── operations/
├── presentations/
├── security/
├── website/
├── ASSET-001_Asset_Library_Standard.md
└── README.md

---

## 4. Naming Convention

Use lowercase.

Separate words with hyphens.

Examples:

architecture-package-v0.5-release.png

network-topology-v1.0.png

github-homepage.png

project-orion-homepage.png

hero-banner-v2.png

---

## 5. Image Formats

PNG for screenshots.

SVG when available for diagrams and logos.

JPEG only for photographs.

---

## 6. Reuse Policy

Assets should be referenced from the Asset Library.

Duplicate copies should be avoided whenever practical.

---

---

## 8. Guiding Principles

The Project Orion Asset Library follows these principles:

- Single source of truth
- Reuse before duplication
- Organized by engineering domain
- Consistent naming conventions
- Version-controlled assets
- Documentation-first engineering
